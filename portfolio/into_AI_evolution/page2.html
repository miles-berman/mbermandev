Part 2: Why We Need More

Going beyond binary

Once our problem becomes something more complex, we need to go beyond just a randomly generated population. Right now, we're just ignoring all the organisms that pass the challenge. This will in theory work... eventually. But we can do better. Let's start with an example. Using basically the same organisms we coded before,we can introduce a new node that generates a random number between 0-10. Let's now remove all the organisms that aren't a 1. This means our problem has changed from a binary problem to a unary problem.

The code to make a random int instead:


# Python code for a random 0-10 class

# RandNode
class RandNode:
    def __init__(self, value=0):
        self.data = random.randint(0, 10) # random value between 0 and 10

    def forward(self, value):
        self.data = value

All we do here is simply change the hardcoded 0/1, we generate the data of the node to a random int.



Repopulating with children instead

When dealing with more complex problems, a more nuanced approach is required. We are now looking to populate our grid with offspring rather than random new organisms. This introduces the concept of inheritance and genetic algorithms into our simulation.
